  • Parent calls calculateLayout() on child panels, not triggered manually
Key Design Principles

Prevent Visual Jumping:

  • Sizing Rule: Never use self.width / 2 inside createChildren. Use addColumnFill() and let the layout engine calculate at runtime.
  • Nesting Rule: Every ISPanel used as a container must have its calculateLayout called by the parent, never manually inside its own logic.
  • Anchoring Rule: Use "Spring" rows (rows with no minimum height but "Fill" enabled) to push buttons to the bottom.
