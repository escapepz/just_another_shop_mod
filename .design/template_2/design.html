✓ Key Improvements Over Template 1

1. Multi-Listing Architecture

Owner can set multiple exchange options for one item. Instead of prices[item] = single_price, use listings[item] = [list_of_options]

2. Dynamic Grid Responsiveness

Grid calculates columns at runtime: floor(availableWidth / tileWidth). Scales from 2 to 5+ columns without hardcoding.

3. Real-Time Item Validation

As owner types item ID, system validates immediately and shows icon. Prevents invalid entries and gives instant feedback.

4. 3-Section Form Layout

Draft inputs, active listings, and footer actions clearly separated. More professional and easier to navigate than simple 2-row layout.

5. Enhanced State Management

Separate ownerUI and customerUI state objects. Track draft listings, selections, and validation errors independently.

6. Advanced Error Handling

Validation feedback shown inline (red borders, error messages). Button states respond to validation (disable if constraints violated).

Design Principles (Enhanced)

Core Rules to Prevent Visual Jumping

  • Sizing: Use addColumnFill() or addColumn(0.5), never hardcoded self.width / 2
  • Grid Responsiveness: Calculate column count at render time: columnCount = floor(width / tileWidth)
  • Nesting: Parent always calls calculateLayout() on child panels—never internally triggered
  • Anchoring: Use Spring rows (addRow(0) with Fill) to push buttons to bottom
  • Validation: Show feedback immediately on input, don't wait for button click
  • Scrolling: Use ISScrollingListBox for lists exceeding fixed height to prevent UI overflow
Data Structures & State

Shop Listings (modData)

Store multiple barter options per item:

modData.shopListings = {
  ["Base.Water"] = {
    {exchangeFor = "Base.Nails", count = 100, id = "listing_001"},
    {exchangeFor = "Base.Hammer", count = 1, id = "listing_002"}
  },
  ["Base.Food"] = { ... }
}

Session State Objects

-- Owner UI State
ownerUI = {
  selectedItemInInventory = "Base.Water",
  draftListing = {exchangeFor = "Base.Nails", count = 100},
  activeListings = { ... },
  validationError = nil
}

-- Customer UI State
customerUI = {
  selectedItemInShop = "Base.Water",
  availableBarterOptions = { ... },
  playerCanAfford = true
}
