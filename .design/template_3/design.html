Template 3: Unit-Based Barter System

Bidirectional Trading with Configurable Offers and Requests

Core Concept: Bidirectional Transactions

Template 3 introduces a fundamental shift from one-way pricing to true bartering. Instead of "customers pay fixed price for items," the shop now operates on transaction pairs: "Give X units of my item in exchange for Y units of their item."

Example: Owner has 12 water bottles and can offer:
• 1 bottle for 100 nails
• 2 bottles for 1 baseball bat
• 3 bottles for 50 ammo
Each bundle size is configurable independently.

35/65 column split focuses on transaction building. Owner selects an inventory item, then defines multiple "Give X, Get Y" trade pairs. The interface validates stock limits in real-time.

Owner View Features

  • Bidirectional Trading: Configure both OFFER (what shop gives) and REQUEST (what shop wants) independently
  • Dynamic Stock Validation: "You have 12 available" updates in real-time based on selected quantity
  • 35/65 Column Split: Narrower left inventory panel, wider form for detailed transaction building
  • Visual Trade Summary: Arrow-based card shows both sides of transaction before confirming
  • Multiple Bundles: Owner can offer same item in different quantities (1×, 2×, 3×) with different prices
  • Real-Time Validation: Add button disabled if REQUEST item invalid or OFFER exceeds stock

Customer View Features

  • Trade Selection Interface: Radio buttons or clickable cards show all available bundles for selected item
  • Visual Arrow Representation: `1× → 100×` format instantly shows what's being traded
  • Affordability Checking: Shows what player needs, has, and whether the trade is possible
  • Stock Availability: Displays "Available: 12 trades max" based on shop inventory and offer quantity
  • Contextual Button Text: "Accept Trade: Give 1× Bottle, Get 100× Nails" clarifies the transaction
  • Real-Time Updates: As customer changes selections, affordability and availability recalculate

Aspect Template 2 Template 3 Improvement

Column Layout 50/50 split 35/65 split (narrower form) Less overwhelming UI

Customer Choice Single action per item Choose from multiple bundle options Better user agency
Design Principles (Advanced)

Unique to Template 3

  • Bidirectional Validation: Both OFFER and REQUEST must be valid. OFFER can't exceed inventory
  • Dynamic maxRepeats: Calculated as `floor(inventoryQty / offerQty)` for each trade
  • Transaction Pair Concept: Every trade is a complete unit (offer + request), not separate fields
  • Arrow Visual Language: Use `→` to represent flow of transaction (offer → request)
  • Column Ratio Strategy: 35/65 prioritizes form over inventory since transactions are the focus
  • Bundle Scaling: If owner has 12 items, offer 1× = 12 trades, offer 2× = 6 trades, offer 3× = 4 trades

Template 3 represents a mature, feature-complete barter system. The architecture supports complex trading scenarios while maintaining clean, intuitive UX.
